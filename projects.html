  • 👩‍💼 My role: Implemented EntityManager, contributed to UML & Frenzy Mode logic

OOP & Design Patterns:

  • SOLID principles applied to input/output handling and scene abstraction

📊 PaperTrades – Mobile App for Simulated Stock Trading

INF2003 Database Systems | Role: Backend & Performance Analyst

PaperTrades simulates stock trading with real data, helping students understand portfolio building. Built using Flutter and Node.js with both NoSQL and SQL databases.

  • 🧪 Tech Stack: Flutter, Node.js, MongoDB Atlas, Supabase (PostgreSQL), Azure
  • 👩‍💼 My role: NoSQL migration, backend integration, performance benchmarking
System Highlights:
  • Live stock price feed via Finnhub
  • Dockerized backend on Azure
  • Hybrid database architecture: SQL for User, NoSQL for transactions/assets
Performance Findings:
  • Indexed MongoDB matched SQL performance (~1ms vs 0.2ms)
  • Reduced memory usage with optimized aggregation queries
